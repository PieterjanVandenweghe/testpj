Concluding the task: <example>, <result> and <postreq>

You can conclude your task topics by adding an example (<example>), describing the outcome of the task (<result>) or pointing out what the user has to do next (<postreq>). Only insert such a closing element when they have an added value for the users. For example, it is meaningless to add the <result> "You have now created a new topic" to the topic "How to create a new topic".

<example>

After you described the task, you can include an example that illustrates the entire task. Well-chosen examples are often more informative than the written steps.

Tip: To make the example stand out, you can insert a <title> element in the <example> element. Avoid using the vague "Example" as title, try to be more specific.

<result>

In the <result> element, you can describe the expected result(s) of the task or specify what occurs after the task is completed. Be careful, however, not to describe the obvious: the result of the task is usually clear from the title.

<postreq>

You can use the <postreq> element to describe what users must do after they completed the task. This is especially useful when the task is part of a larger procedure. You can then use the <postreq> element to guide the users to the next step in that procedure.
