Organizing the information in tables: <table>, <simpletable> and <properties>

Tables are by far the best way to structure reference information. They make it much easier for users to scan the information and to find the item they are looking for. In a reference topic, you can use common tables (<table> element), simple tables (<simpletable> element) and properties tables (<properties> element).

<table>

Most tables are common tables, wrapped in a <table> element. You can also add a title to common tables, by inserting a <title> element into the <table> element. If you add a title to your table, avoid using generic titles like "Table 1". Always specify which information the users can find in the table.

<simpletable>

If you do not need a title for your table, you can also use a simple table. For example, a title can be unnecessary when the topic title already accurately describes the content of the table. By not including a title, you avoid repeating the topic title in the table title.

Tip: Instead of using a simple table, you can also use a common table and omit the <title> element.

<properties>

Unlike common tables and simple tables, properties tables are only valid in reference topics. You can use the <properties> element to create a table that lists types, values and descriptions of properties.
