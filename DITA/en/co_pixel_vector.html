Vector graphics and pixel images

In a digital context, you can distinguish two types of graphics: vector graphics and pixel images. You can scale vector drawings to any size without major quality loss, while this is not the case with pixel images.

Vector graphics

Vector graphics consist of a number of lines, connecting points on the screen. The graphics store only the locations of the points and the type of lines connecting them. As a result, when you scale this graphic, the computer can recalculate the locations and line types to fit the new size, without negative effects on graphic quality.

Examples of vector graphics are:
  • eps
  • ai
  • svg
  • pdf
  • cgm

Pixel images

Pixel images consist of a pixel-by-pixel mapping of the graphic on the screen. The images store information on the type (color) of every pixel in the image (for example, 0 for white, 1 for black).

This illustration shows you what a line in a pixel image looks like when you zoom to pixel level:

When you enlarge a pixel image, the information determining what was originally the color of a single pixel suddenly defines multiple pixels, resulting in a loss of sharpness. For example, if you would enlarge a pixel image by 100%, this would result in each original pixel blown up to 4 times its original size.

Examples of pixel images are:
  • jpeg
  • tiff
  • gif
  • png
  • pct
  • bmp
  • img
