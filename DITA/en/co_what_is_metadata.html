What is metadata?

Metadata is information you can add to a DITA map (or a topic) that gives more information about the content. As it is generally information that is more useful to the author than to the reader, the reader does not get to see it. Metadata can include information such as creation date, author, semantics of the textual components, and copyright ownership.

Data vs. metadata

An important distinction should be made between data and metadata. Data is synonymous with content, so the information you write in your topics will be data. Metadata, on the other hand, is information about the content, such as the name of the author, when the files were created, who the copyright holder is etc.

This distinction becomes important when you look at the types of files you will use in DITA. Topics can contain content but DITA maps should never contain content. As such, topics can have data as well as metadata, whereas DITA maps should only ever have metadata.

Overriding metadata

You can create metadata in both topics and maps. The metadata in a topic applies to all the elements within the entire topic. Similarly, the metadata of a DITA map applies to the entire map. As such, it will override the metadata of its topics.

There are ways to prevent this and ensure the metadata in a topic takes precedence. You can, for instance, adjust the @lockmeta attribute on the <topicmeta> element to control whether map-specified values override values in the referenced topic.

Types of metadata

Type Purpose
Critical dates (critdates) In this element you can store, for instance, the date your topic or map was created or was revised.
author This element contains the name of the topic's author and can contain more information about the author, such as contact information, role etc.
copyright This element contains no text, only nested elements which specify the year (copyryear) and the owner of the copyright (copyrholder).
prodinfo This product information element contains no text, only nested elements which specify the product name, version, release date, brand, series, platform etc.
